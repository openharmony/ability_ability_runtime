## 七大检查维度速查

```
┌─────────────────────────────────────────────────────────┐
│  1. 控制流  →  死代码、逻辑矛盾、覆盖不完整              │
│  2. 数据流  →  未初始化、数据污染、类型不匹配            │
│  3. 状态机  →  非法转换、状态不一致、死锁                │
│  4. 边界条件  →  越界、空指针、溢出                      │
│  5. 错误处理  →  遗漏错误路径、资源泄漏                  │
│  6. 并发控制  →  死锁、竞态条件、数据竞争                │
│  7. 业务规则  →  不变性破坏、契约违反                    │
└─────────────────────────────────────────────────────────┘
```

## 常见问题速查表

| 问题类型 | 检测方法 | 严重等级 |
|---------|---------|----------|
| 死代码 | 永不为真的条件、return后的代码 | 🟡 |
| 逻辑矛盾 | 互斥条件同时为真 | 🟠 |
| 条件覆盖不完整 | 枚举/switch缺少分支 | 🟠 |
| 未初始化变量 | 使用前未赋值 | 🔴 |
| 数据污染 | 外部数据未验证 | 🔴 |
| 类型不匹配 | 有符号/无符号比较 | 🟠 |
| 非法状态转换 | 跳过必要中间状态 | 🔴 |
| 状态不一致 | 多状态变量不同步 | 🟠 |
| 状态机死锁 | 无法到达终态 | 🔴 |
| 数组越界 | 索引未验证 | 🔴 |
| 空指针解引用 | 未检查空指针 | 🔴 |
| 整数溢出 | 算术运算溢出 | 🔴 |
| 遗漏错误路径 | 不检查返回值 | 🟠 |
| 资源泄漏 | 错误路径未释放 | 🔴 |
| 死锁 | 锁顺序不一致 | 🔴 |
| 竞态条件 | Check-Then-Act | 🔴 |
| 数据竞争 | 无保护并发访问 | 🔴 |
| 不变性破坏 | 破坏类约束 | 🔴 |
| 契约违反 | 前置/后置条件未验证 | 🟠 |

## 代码模式速查

### ❌ 错误模式

```cpp
// 1. 死代码
if (false) {
    NeverExecuted();  // 🟡
}

// 2. 逻辑矛盾
if (x > 10 && x < 5) {  // 🟠
    Impossible();
}

// 3. 未初始化
int result;
if (condition) {
    result = 10;
}
return result;  // 🔴

// 4. 空指针
Account* acc = GetAccount();
acc->Update();  // 🔴

// 5. 数组越界
for (int i = 0; i <= size; i++) {  // 🔴
    data[i];
}

// 6. 遗漏错误处理
database_->Insert(info);  // 🟠
return ERR_OK;

// 7. 竞态条件
if (instance == nullptr) {  // 🔴
    instance = new Instance();
}

// 8. 非法状态转换
state = DEACTIVATED;  // 从CREATED直接跳过  🔴
```

### ✅ 正确模式

```cpp
// 1. 完整的条件覆盖
if (state == ACTIVE) {
    HandleActive();
} else if (state == INACTIVE) {
    HandleInactive();
} else {
    HandleUnknown();  // ✅
}

// 2. 初始化变量
int result = DEFAULT_VALUE;  // ✅
if (condition) {
    result = 10;
}
return result;

// 3. 空指针检查
Account* acc = GetAccount();
if (acc == nullptr) {  // ✅
    return ERR_NULL;
}
acc->Update();

// 4. 边界检查
for (int i = 0; i < size; i++) {  // ✅
    data[i];
}

// 5. 完整错误处理
ret = database_->Insert(info);  // ✅
if (ret != ERR_OK) {
    return ret;
}

// 6. 线程安全
std::lock_guard<std::mutex> lock(mutex);  // ✅
if (instance == nullptr) {
    instance = new Instance();
}

// 7. 合法状态转换
if (state == ACTIVE) {  // ✅
    state = DEACTIVATED;
}
```

## 严重等级决策树

```
发现问题
  ↓
是否影响系统正确性？
  是 → 🔴 致命 (必须修复)
  否 ↓
是否影响安全性或稳定性？
  是 → 🔴 致命 (必须修复)
  否 ↓
是否可能导致功能异常？
  是 → 🟠 严重 (强烈建议修复)
  否 ↓
是否是最佳实践问题？
  是 → 🟡 警告 (建议修复)
  否 → 🟢 建议 (可选优化)
```

## 快速检查清单

### 30秒快速检查

- [ ] **是否有未检查的返回值？**

### 5分钟详细检查

#### 控制流
- [ ] 无死代码
- [ ] 无逻辑矛盾
- [ ] 条件覆盖完整

#### 数据流
- [ ] 变量已初始化
- [ ] 外部数据已验证
- [ ] 类型匹配

#### 状态机
- [ ] 状态转换合法
- [ ] 状态保持一致
- [ ] 可到达终态

#### 边界条件
- [ ] 数组边界验证
- [ ] 空指针检查
- [ ] 整数溢出检查

#### 错误处理
- [ ] 所有错误路径处理
- [ ] 资源正确释放
- [ ] 错误码正确传递

- [ ] 锁顺序一致
- [ ] 共享变量保护
- [ ] 无数据竞争

#### 业务规则
- [ ] 不变性维护
- [ ] 契约遵守
- [ ] 约束条件验证
